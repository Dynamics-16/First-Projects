JavaScript "Basics"

I have just started to learn JavaScript (27/01/2024). I think it may be a good idea to note some things down from what I learn, this way I get the best of both worlds - I can continue learning JavaScript and also practice my HTML in the process! 


  • <Script></Script>

    This element is used to load Javascript into your HTML file! Don't forget to include the src attribute (i.e. ./script.js).

    The <script> element in HTML can be placed in the <head> section or the <body> section of your HTML document. The placement depends on what you are trying to achieve.


    Here are the considerations for each placement:

    In the <head> section:

    Placing scripts in the <head> section is common when the script contains functions, variables, or other code that needs to be loaded before the rest of the page.

    This is often the case when you want to define functions or variables that are going to be used throughout your entire document.


    Before the closing </body> tag:

    Placing scripts just before the closing </body> tag is common when you want to ensure that the HTML content is loaded before the scripts are executed.

    This can improve the perceived page load time because the browser doesn't have to wait for the scripts to be downloaded and executed before rendering the page.


  • var

    In JavaScript a variable is used to hold a value. To use a variable you must first declare it! For example:

    var camperbot;

    As you can see, camperbot is the variable!

    Variables can be assigned a value. When you do this while you declare it, this is called initialization. For example:

    var camperbot = "Bot";

    This would initialize the camperbot variable with a value of Bot, a string.

    Do note: When a variable name has multiple words, the convention in JavaScript is to use what's called camelCase. The first word is lowercase, and the first letter of every following word is uppercase.


  • let

    In modern JavaScript, it's best practice to use the keyword let instead of the keyword var.

    This fixes several unusual behaviors with var that can make your code difficult to debug.


  • const

    If you are not going to assign a new value to a variable, it is best practice to use the const keyword to declare it instead of the let keyword.

    This will tell JavaScript to throw an error if you accidentally reassign it.


  • Strings

    JavaScript has multiple different data types. Strings are used to store things like words or text. Strings are surrounded with double quotes, single quotes, or backticks.

    Here is an example of declaring a variable with a string:

    let developer = "Naomi";

  • Arrays

    An array can be used to hold multiple values. For example:

    let order = ["first", "second", "third"];

    This is an array which holds three values. Notice how the values are separated by commas.


  • getElementById()

    The getElementById method can "find" an HTML element and can change the element content.

    It can also change HTML attribute values, alter CSS styles and hide/show html elements!

    If you scroll down below, you can see a few examples of this!


  • window.alert()

    The window.alert() method is used to display a dialog box with a specified message and an 'OK' button

    The 'window' object represents the global browser window or tab, and the 'alert' is a method attached to this object!
